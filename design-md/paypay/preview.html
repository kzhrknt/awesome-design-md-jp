Layout

レイアウト原則

HERO: 大型グラデーション + Noto Sans JP 55px 700 / lh ×1.6 / 中央寄せ
FONT SWITCHING: body は Hiragino ProN、見出しは Noto Sans JP に切り替える二段構成
BUTTON RADIUS: 6px が PayPay 共通（UNIQLO の 0、Nintendo の 6 とは異なる中間値）
HEADING WEIGHT: 全見出し 700 統一 / サイズと色（黒 vs 赤）でメリハリ
LH RATIO: 見出し ×1.6 / 本文 ×1.4 / リード ×1.6〜2.0
Font Note

フォントの注記

PayPay は和文の本文に 「Hiragino Kaku Gothic ProN」（ProN — JIS90 字形） を最先頭に置きます。UNIQLO・Apple Japan の Pro（JIS78 字形）とは別物で、字形の好みが異なります。

主要見出し（H1〜H3）では Noto Sans JP に切り替え、ヒラギノを後段に置く二段構成。見出しウェイトは 全 700 統一で、サイズとカラー（黒 #242323 vs 赤 #F24F4F）でメリハリを付けます。

"SCROLL" 等の Display 用英字に Futura が指定されているのも特徴的です。

PayPay 独自の CSS Custom Properties は事実上存在しません。:root に観測される 47 個の --wp--preset-* は WordPress Gutenberg のデフォルト値であり、PayPay のデザイントークンではない点に注意。デザイントークンは BEM 形式の class 名（mainButton__button, campaign__title 等）に直接エンコードされています。
