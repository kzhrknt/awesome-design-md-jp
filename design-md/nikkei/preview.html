レイアウト原則

HEADER: 二段構成 / 上段: ロゴ + ツール + ログイン/購読 CTA / 下段: ナビゲーションタブ（Navy 背景）
HERO ARTICLE: 27px トップ記事 + 引き出し + メタ情報 + 下に 2px 太線の区切り
ARTICLE GRID: 3 カラム / 各カラムにジャンル別記事を縦積み / 罫線で区切る新聞紙面風
SIZE HIERARCHY: 27 / 24 / 22 / 20 / 18 / 16 / 14 / 12 の8段階を駆使し、サイズ差で情報優先順位を視覚化
FONT STRATEGY: Web フォントを使わず、OS システム UI フォント（-apple-system / system-ui / Segoe UI）に任せる
NO LETTER-SPACING: 全要素 normal — 新聞らしいベタ組徹底
Font Note

フォントの注記

日経電子版の最大の特徴は、和文フォントを明示しないシステム UI フォント主体のスタックを採用していることです。-apple-system, "system-ui", "Segoe UI", Roboto, Oxygen, ... という多 OS のシステムフォント混在 stack で、日本語は OS 標準（macOS の Hiragino、Windows の游ゴシック等）に任せています。

この設計の利点:

  • Web フォントを読み込まない = ファーストペイント高速化、FOIT/FOUT 完全回避
  • 各 OS のネイティブな読書体験を尊重
  • 新聞らしい中立性を保つ

レガシー領域（ヘルプページ等）には伝統的な "ヒラギノ角ゴ Pro W3", "Hiragino Kaku Gothic Pro", "ＭＳ Ｐゴシック" 指定が残ります。

letter-spacing は全要素 normal。新聞らしいベタ組徹底で、デザインの装飾性を完全に排除。情報密度と可読性を最優先する設計判断です。
